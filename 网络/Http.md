# HTTP

## HTTP1.0/1.1
* 协议：TCP/IP
* 弊端：队头阻塞（源于发送端和接收端的窗口控制--流量控制，TCP协议的连续性导致）
http1.1 的改进：pipelin(一个TCP连接可以发送多个请求)，一个域名最多同时开启6个TCP连接, 没有解决对头阻塞问题

## HTTP2
* 协议：TCP/IP TLS加密
HTTP2的改进：多路复用，服务端推送，首部压缩，二进制流
* 多路复用: 一个TCP连接可以同时发送多个请求，在发送端聚和，在接收端拆分，将数据拆分为一个个Frame,一组Frame组和成一个Stream, Stream作为TCP的传输单元，这里就会产生同样的队头阻塞问题，当多个Stream同时发送的时候，第一个Stream中的一个Frame发送失败，这就导致需要重传，后续的已经完成的Stream也无法被处理

## HTTP3

* 协议：QUIC
QUIC协议：基于UDP的传输层协议，最少可以达到0RTT, QUIC协议是属于传输层协议，QUIC在UDP的基础上基于TLS1.3协议对数据进行加密，保证数据传输的安全性，同时QUIC实现了重传机制确认机制，保证数据的完整性，同时还支持多路复用，首部压缩等。

* QUIC的优点：QUIC建立的连接不像TCP那样是基于IP的而是通过唯一的连接ID来确认连接的，所以这就可以在使用端无缝切换网络。
